## 💫 项目愿景
在嘈杂且过度金融化的社交时代，Semaphore 致力于为同频者建立一个“避风港”。我们通过去中心化技术，守护社交中的隐私、仪式感与精神共鸣，让每一次连接都成为链上永恒且纯粹的“信号”。

---

## 🎯 项目定位
Semaphore 既是古老的“旗语”，也是以太坊生态中著名的 ZK（零知识证明）协议库。

这一命名精准定义了我们的产品灵魂：在技术底层，产品承袭 ZK 协议“匿名信号传递”的硬核精神，实现去中心化的隐私守护；在应用表层，我们将其重构为一种极具仪式感的“精神旗语”空间。

这不仅是一套私密社交系统，更是一个利用 Lit Protocol 等 Web3 原生技术，在暗夜中寻找同频共鸣的“数字灯塔”。它让创作者在完全掌控数据主权的前提下，发射信号，被懂得的人看见、照亮并彼此连接。

| 维度 | 传统社交 (X / Threads) | 创作平台 (Mirror / Medium) | Semaphore |
| --- | --- | --- | --- |
| **核心驱动** | 算法与流量 | 内容资产化 | **精神共鸣** |
| **隐私等级** | 平台可见 / 弱隐私 | 链上完全透明 | **加密授权访问 (Lit)** |
| **用户压力** | 高(点赞/转发数) | 中(收益预期) | **极低(限时消失/私密)** |
| **关系属性** | 弱连接 / 粉丝关系 | 经济利益连接 | **深层同频 / 精神图谱** |

## 🔍 核心痛点与解决方案
### 1. 社交主权的缺失 vs. 去中心化存储
- 痛点：传统平台掌控数据，私密内容随时可能被删除或审阅。
- 方案：信号内容通过 IPFS/Arweave 永久存储。利用 Lit Protocol 实现去中心化访问控制，内容的“钥匙”完全由创作者定义的链上条件（如持有特定凭证）掌握，实现真正的“内容主权”。

### 2. 社交噪音与内卷 vs. 低压力交互逻辑
- 痛点：算法驱动的流量竞争导致创作焦虑，社交压力巨大。
- 方案：没有流量驱动，采用“信号匹配”机制。利用智能合约确保“限时阅读”，将社交从“数据竞赛”回归到“仪式感连接”。

## ✨ 产品核心创新
### 1. 信号传递链：链上精神图谱
Semaphore 不仅是孤立的文字，更是一条由共鸣织就的关系网络：
- 共鸣溯源：每一个后续信号都指向前序交易哈希，在链上构建出透明的“回声链”。

### 2. 微光经济模型
我们设计了一套非侵略性的激励机制，让价值在静谧中流动：
- 微光星尘：读者可用微量代币（如 0.01 AVAX）进行“打赏”，前端表现为信号亮度的柔和提升，而非冰冷的数字。
- 能量守恒机制：发布信号需消耗“能量”，而获取能量的唯一方式是“倾听并回应”他人的信号。这迫使社区形成“先倾听、后发声”的良性循环。
- 自动化收益共享：若信号触发长链传递，后续产生的“星尘”将通过智能合约自动分润给链条上的每一位共鸣者。

## 🛠️ 技术栈
- 网络：Avalanche Fuji Testnet（初期部署）
- 访问控制：Lit Protocol（门限加密解密）
- 存储：IPFS / Arweave
- 前端：React + Vite（Tailwind CSS 氛围渲染）
