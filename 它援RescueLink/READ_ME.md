# 🐾 它援 RescueLink丨基于 Avalanche 的流浪动物救助协同协议

**连接善意，守护生命。**
将流浪动物救助从个人孤立行动，转化一个可协作、可追踪、可留痕、可激励的透明闭环。


## 🚨 Background | 背景与痛点

当前流浪动物救助面临严重的信任黑箱与协作孤岛问题：
* **信息碎片化：** 救助信息、进展与凭证散落在朋友圈、小红书，交接效率极低，责任边界模糊，救助链路极易因信息断层而中断。
* **信任成本高：** 凭证易丢失，捐助者难以验证 case 真实性与实际缺口，资金流向不透明。
* **救助人负担：** 不同参与方之间责任边界不清，一线救助人长期承受极高的经济与精力压力。
* **协作低效：** 送医、转运、回访等关键隐形贡献缺乏记录与反馈，救助体系高度脆弱。

> “针对这四个核心痛点，RescueLink 利用 Avalanche 的底层能力，构建了一套透明、标准、且具备正向激励的协同协议。”

## 🚀 Key Features丨主要功能

### 1. 📋 结构化救助档案 (Structured Case Dashboard)
* **Case 唯一身份:** 为每一个救援案例生成专属页面，整合动物档案、紧急程度及实时缺口。
* **实时进度追踪:** 动态展示“发现-送医-寄养-领养”的全生命周期状态。
* **多方协同入口:** 支持志愿者、医生、捐助者在同一页面下进行角色化交互，告别私聊维系。

### 2. 🔐 链上节点式存证 (On-chain Milestone Tracking)
* **关键摘要上链:** 案件创建、交接确认、结案状态等核心节点同步至 Avalanche 链上。
* **凭证哈希化:** 医疗单据、付款截图等通过哈希存证，确保原始数据在保留隐私的同时，具备防篡改的核验基础。
* **透明流转:** 捐助者可直观查看“善意”如何转化为具体的医疗服务或物资。

### 3. 🛡️ 节点式资金核验机制 (Milestone Disbursement)
* **定向支付逻辑:** 资金与特定 Case 绑定。医疗类支出优先支持定向支付至合作机构，最大程度降低资金误用风险。
* **分阶段释放:** 善款根据救助进展分批拨付，每一笔释放均需匹配相应的链上凭证，实现风控前置。

### 4. 🎖️ 激励与信用体系 (Incentive & Reputation)
* **贡献可视化:** 记录志愿者的送医、回访等行为，生成可核验的参与轨迹。
* **灵魂绑定徽章 (SBT):** 赋予长期活跃参与者“信誉标识”，打造民间救助的信用档案。

## 🔧Tech Stack丨技术栈 
* **前端:** React 19 / Next.js: 利用 SSR (服务端渲染) 优化救助案例的社交分享预览与首屏加载速度。TypeScript: 强类型驱动，确保救助状态机（发现-送医-领养）逻辑的严密性与可维护性。
* **UI/UX 设计 :** Tailwind CSS: 原子化样式管理，实现高性能、响应式的 Mobile-First 救助现场操作界面。Web3 Aesthetic: 采用 Glassmorphism (玻璃拟态) 与深色模式，提升在高对比度户外场景下的阅读体验。
* **数据与存储 :** Supabase / Node.js: 提供实时数据同步 (Real-time DB)，实现多方志愿者协作状态的即时刷新。IPFS (Pinata): 去中心化内容寻址存储，确保医疗票据、检查报告等核心凭证的永久性与不可篡改性。
* **协议与基础设施:** Avalanche (Fuji Testnet): 利用其 Sub-second Finality (亚秒级确认) 特性，提供极致丝滑的链上交互存证体验。Solidity: 核心业务合约，实现 CaseRegistry (案例管理) 与 SBT (身份勋章) 逻辑。Wagmi / Viem/MetaMask: 基于 React Hooks 的轻量化方案，管理 MetaMask 钱包连接与合约调用。
